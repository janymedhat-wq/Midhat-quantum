WAVE EQUATION HPC

1D Vibrating String Simulation: Finite Difference Method

A guide to modeling the classical motion of a perfectly elastic string, fixed at both ends, using a high-fidelity numerical scheme.

The motion of a small transverse displacement $u(x, t)$ along an ideal, tensioned string is governed by the **1D Wave Equation**. This hyperbolic partial differential equation is the foundation for understanding wave propagation in classical physics and acoustics.

The 1D Wave Equation

$$\frac{\partial^2 u}{\partial t^2} = c^2 \frac{\partial^2 u}{\partial x^2}$$

Where $c$ is the wave speed, determined by the string's properties: $c = \sqrt{T/\rho}$, with $T$ being the tension and $\rho$ being the linear mass density. For the fixed-end simulation, the boundary conditions are: $u(0, t) = 0$ and $u(L, t) = 0$.

1. Numerical Method: FDTD Scheme

To solve the wave equation numerically, the **Finite Difference Time Domain (FDTD)** method is commonly used. This involves discretizing both the spatial domain ($x$) and the temporal domain ($t$) and approximating the derivatives with finite difference quotients.

The Central Difference Approximation

Using a central difference scheme for both time and space, we arrive at the explicit update formula for the displacement $u$ at the next time step $n+1$:

$$\frac{u_{i}^{n+1} - 2u_{i}^{n} + u_{i}^{n-1}}{(\Delta t)^2} = c^2 \frac{u_{i+1}^{n} - 2u_{i}^{n} + u_{i-1}^{n}}{(\Delta x)^2}$$

Rearranging this equation yields the core update step:

$$u_{i}^{n+1} = 2u_{i}^{n} - u_{i}^{n-1} + \lambda^2 (u_{i+1}^{n} - 2u_{i}^{n} + u_{i-1}^{n})$$

where $\lambda = c \Delta t / \Delta x$ is the **Courant-Friedrichs-Lewy (CFL)** number. For numerical stability, we must ensure $\lambda \le 1$.

2. Parallel Implementation (C++ / OpenMP)

The FDTD scheme is inherently local: the value of $u$ at a point $i$ depends only on its immediate neighbors ($i-1, i, i+1$) from the previous time step. This locality makes the method highly suitable for parallelization, typically using OpenMP for shared memory or MPI for distributed memory on large clusters.

// string_fdtd.cpp: Conceptual FDTD time-stepping
#include 
#include 
#include 

// Constants and Parameters
const double C_SQUARED = 1.0; // c^2 = T/rho
const double DX = 0.01;      // Spatial step size
const double DT = 0.005;     // Time step size
const double LAMBDA_SQ = C_SQUARED * (DT * DT) / (DX * DX); // CFL condition must hold (lambda <= 1)

void update_string_step(
    const std::vector& u_current, // u(t)
    const std::vector& u_previous, // u(t - dt)
    std::vector& u_next)          // u(t + dt)
{
    int N = u_current.size();

    // The boundary points (index 0 and N-1) are fixed (u=0)
    u_next[0] = 0.0;
    u_next[N - 1] = 0.0;

    // OpenMP/CUDA Parallel region typically starts here.
    // Each thread calculates u_next[i] independently.
    for (int i = 1; i < N - 1; ++i) {
        // Core FDTD Update: u_i^(n+1) = 2*u_i^n - u_i^(n-1) + lambda^2 * (u_{i+1}^n - 2*u_i^n + u_{i-1}^n)
        u_next[i] = 
            2.0 * u_current[i] - u_previous[i] + 
            LAMBDA_SQ * (u_current[i + 1] - 2.0 * u_current[i] + u_current[i - 1]);
    }
}

// End of Conceptual FDTD Core.

3. Initial Conditions: Plucked String

A common starting configuration is the "plucked string" profile, where the string is pulled to a triangular shape and released from rest.

Initial Displacement $u(x, 0)$ and Velocity $\partial u / \partial t |_{t=0}$

The initial displacement $u(x, 0)$ is often defined piecewise, while the initial velocity $\partial u / \partial t |_{t=0}$ is zero, which is essential for initiating the motion using the two-step FDTD method. 

# string_initial_condition.py: Plucked string initialization
import numpy as np

# --- System Parameters ---
L = 1.0             # String length
A = 0.05            # Maximum initial amplitude
P = 0.25            # Plucking point (as fraction of L, here at L/4)
N = 1000            # Number of points

# Create the spatial grid
x_grid = np.linspace(0.0, L, N)
pluck_x = P * L

def plucked_string_initial(x, L, A, pluck_x):
    """Defines the initial triangular displacement profile."""
    u0 = np.zeros_like(x)
    for i, xi in enumerate(x):
        if xi <= pluck_x:
            # Linear rise from 0 to A
            u0[i] = (A / pluck_x) * xi
        else:
            # Linear drop from A to 0
            u0[i] = (A / (L - pluck_x)) * (L - xi)
    return u0

# Initial displacement u(x, 0)
u_t0 = plucked_string_initial(x_grid, L, A, pluck_x)
# Initial velocity du/dt(x, 0) is zero, so u(t=dt) is set using a special case of FDTD
u_t1 = u_t0.copy() 

print(f"Initialized plucked string (L={L}, Max Amp={A}) at P={P}L.")
print("Two state vectors (u_t0 and u_t1) are required to start the FDTD loop.")
# End of Python Initialization.
